Introduction

API REST construite avec Hono dans Next.js 15, base PostgreSQL via Prisma.

Base URL

/api

Local : http://localhost:3001/api

Format réponse

Toutes les réponses suivent :

{ "success": bool, "data": T | null, "message": string }

Audiences

  • Public (apps/front) — x-api-key + x-session-id
  • Admin dashboard — cookie de session better-auth

CORS autorisé

  • http://localhost:3000
  • http://localhost:3001
  • + domaines de production via env

Authentification

Deux mécanismes coexistent selon l'appelant.

Session Cookie (dashboard)

Cookie HTTP-only posé par better-auth via /api/auth/[...all].

Niveaux : requireAuth → requireMonitor → requireAdmin

API Key (front public)

Header x-api-key: <clé>

Pour le panier, ajouter aussi x-session-id: <uuid> (TTL 24h)

Middleware — hiérarchie

MiddlewareVérification
requireAuthSession better-auth valide (tout rôle)
requireAdminSession + rôle ADMIN
requireMonitorSession + rôle MONITEUR ou ADMIN
requireApiKeyHeader x-api-key valide
requireCartSessionHeader x-session-id (crée session si inexistante)

Erreurs & Codes HTTP

Format d'erreur standard.

CodeCas
200Succès (GET, PATCH, PUT)
201Création réussie (POST)
400Validation Zod échouée — body retourne les erreurs de champ
401Non authentifié
403Rôle insuffisant
404Ressource introuvable (Prisma P2025)
409Conflit — contrainte unique (Prisma P2002)
500Erreur serveur interne

Panier côté front. Nécessite x-api-key + x-session-id. Les items expirent après 1h.

HeaderDescription

Réponse 200

{
  "success": true,
  "data": {
    "items": [
      {
        "id": "...",
        "type": "STAGE",
